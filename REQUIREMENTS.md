## 1. 项目背景

### 1.1 问题陈述

随着城市化进程加速，深圳市龙华区面临严重的停车难题：
- **停车位供需失衡**：核心商圈停车位紧张，外围停车场利用率低
- **交通拥堵严重**：车辆寻找停车位造成无效交通流量
- **信息不对称**：驾驶员无法实时获取停车场可用信息
- **资源分配不均**：缺乏智能调度机制，停车资源浪费

### 1.2 项目目标

构建一个智能停车协同疏导系统，实现：
- **智能推荐**：基于位置、价格、可用性的智能停车推荐
- **分级调度**：三级车场池（核心商圈/外围配套/路边临停）分级管理
- **实时感知**：通过City Brain API实时获取停车场状态
- **无缝体验**：从寻车到支付的一站式停车服务

### 1.3 项目范围

**包含功能**：
- 车辆实时监控与路径规划
- 智能停车推荐系统
- 三级车场池分级管理
- City Brain停车API集成
- 在线支付处理
- 实时数据推送
- 管理后台接口

**不包含功能**：
- 移动APP开发（当前为Web版本）
- 硬件传感器集成
- 第三方导航SDK深度集成
- 企业级用户管理系统

## 2. 功能需求

### 2.1 车辆管理模块

#### 2.1.1 车辆创建
- **需求描述**：管理员可以创建车辆实体，设置初始位置和目的地
- **输入参数**：
  - 车辆ID（可选，系统自动生成）
  - 初始位置（经纬度）
  - 目的地（经纬度或地址）
  - 车辆类型
- **输出结果**：车辆创建成功，返回车辆完整信息
- **业务规则**：
  - 车辆ID全局唯一
  - 位置坐标必须在有效范围内
  - 目的地不能与初始位置相同

#### 2.1.2 车辆查询
- **需求描述**：用户可以查询所有车辆状态或单个车辆详情
- **输入参数**：车辆ID（可选）
- **输出结果**：车辆列表或单个车辆详情
- **业务规则**：
  - 支持分页查询
  - 实时返回车辆位置和状态

#### 2.1.3 路径规划
- **需求描述**：为车辆规划从当前位置到目的地的最优路径
- **输入参数**：
  - 起点坐标
  - 终点坐标
  - 路径偏好（最短/最快/避开拥堵）
- **输出结果**：路径信息（距离、时间、路线详情）
- **业务规则**：
  - 集成高德地图API
  - 支持实时路况
  - 路径缓存机制

#### 2.1.4 实时同步
- **需求描述**：车辆状态变更时实时推送给前端
- **技术实现**：WebSocket长连接
- **推送内容**：
  - 位置更新
  - 状态变更
  - 路径变更
- **业务规则**：
  - 自动重连机制
  - 心跳保活
  - 断线重传

### 2.2 智能停车推荐模块

#### 2.2.1 停车搜索
- **需求描述**：根据用户位置和偏好搜索附近停车场
- **输入参数**：
  - 用户位置（经纬度）
  - 最大距离
  - 最高价格
  - 偏好设置（有盖、充电桩、24小时等）
- **输出结果**：推荐停车场列表，按综合评分排序
- **评分算法**：
  ```
  Score = (DistanceWeight * DistanceScore) +
          (PriceWeight * PriceScore) +
          (AvailabilityWeight * AvailabilityScore) +
          (FeatureWeight * FeatureMatchScore)
  ```
- **业务规则**：
  - 距离权重：50%
  - 价格权重：20%
  - 可用性权重：20%
  - 特性匹配权重：10%

#### 2.2.2 停车位预约
- **需求描述**：用户可以预约特定停车位的特定时间段
- **输入参数**：
  - 停车场ID
  - 车位ID
  - 开始时间
  - 结束时间
  - 用户ID
- **输出结果**：预约确认信息
- **业务规则**：
  - 时间段不能与现有预约冲突
  - 预约时长限制（最长24小时）
  - 预约提前量限制（最多提前7天）
  - 支持预约取消

#### 2.2.3 停车会话管理
- **需求描述**：管理车辆进入和离开停车场的会话
- **输入参数**：
  - 停车场ID
  - 车位ID
  - 车辆ID
- **输出结果**：会话ID、开始时间、预计费用
- **业务规则**：
  - 自动计算停车费用
  - 支持超时费用计算
  - 会话历史记录

### 2.3 三级车场池分级模块

#### 2.3.1 车场池分类
- **需求描述**：将停车场自动分类到三级车场池
- **分类标准**：
  - **核心商圈池（Priority 1）**：
    - 距离中心≤2km
    - 周转率≥5/hour
    - 高端定位
  - **外围配套池（Priority 2）**：
    - 距离中心2-5km
    - 周转率≥2/hour
    - 容量充足
  - **路边临停池（Priority 3）**：
    - 距离中心>5km
    - 周转率<2/hour
    - 临时停车位
- **中心坐标**：深圳市龙华区中心 (22.6913, 114.0448)

#### 2.3.2 车场池推荐
- **需求描述**：基于车场池优先级推荐停车场
- **推荐策略**：
  1. 优先推荐核心商圈池
  2. 核心池无可用车位时推荐外围配套池
  3. 兜底推荐路边临停池
- **评分算法**：
  ```
  Score = (DistanceScore * 0.5) +
          (AvailabilityScore * 0.3) +
          (TurnoverScore * 0.2)
  ```

#### 2.3.3 交通分流触发
- **需求描述**：当区域车流密度超过阈值时触发交通分流
- **触发条件**：
  - 区域车流密度≥阈值（可配置）
  - 目标车场池有可用车位
  - 分流距离在最大范围内
- **分流规则**：
  - 从核心池分流到外围池
  - 从外围池分流到路边池
  - 支持多级分流

#### 2.3.4 车场池统计
- **需求描述**：实时统计各车场池的状态
- **统计指标**：
  - 占用率
  - 周转率
  - 平均等待时间
  - 当前车辆数
- **更新频率**：实时更新

### 2.4 City Brain API集成模块

#### 2.4.1 车辆入场上报
- **需求描述**：向City Brain系统上报车辆入场信息
- **上报内容**：
  - 停车场ID
  - 车牌号
  - 入场时间
  - 车位ID
- **加密方式**：SM4加密
- **签名验证**：MD5签名

#### 2.4.2 车辆出场上报
- **需求描述**：向City Brain系统上报车辆出场信息
- **上报内容**：
  - 停车场ID
  - 车牌号
  - 出场时间
  - 停车费用
- **加密方式**：SM4加密
- **签名验证**：MD5签名

#### 2.4.3 心跳保活
- **需求描述**：定期向City Brain系统发送心跳
- **心跳间隔**：60秒
- **心跳内容**：系统状态、时间戳

#### 2.4.4 Mock服务
- **需求描述**：提供Mock服务用于本地测试
- **Mock功能**：
  - 模拟车辆入场/出场
  - 模拟车位状态更新
  - 模拟心跳响应

#### 2.5.2 支付状态查询
- **需求描述**：查询支付订单状态

## 6. 数据模型

### 6.1 核心实体

#### ParkingLot（停车场）
```go
type ParkingLot struct {
    ID              string    // 停车场ID
    Name            string    // 停车场名称
    Address         string    // 地址
    Latitude        float64   // 纬度
    Longitude       float64   // 经度
    TotalSpaces     int       // 总车位
    AvailableSpaces int       // 可用车位
    PricePerHour    float64   // 每小时价格
    Features        []string  // 特性（充电、有盖等）
    Rating          float64   // 评分
    Distance        float64   // 距离
    IsOpen          bool      // 是否营业
    LastUpdated     time.Time // 最后更新时间
}
```

#### ParkingPool（车场池）
```go
type ParkingPool struct {
    ID           string           // 车场池ID
    Level        ParkingPoolLevel // 级别（core/peripheral/roadside）
    Name         string           // 名称
    Description  string           // 描述
    Priority     int              // 优先级（1=highest）
    Lots         []*ParkingLot    // 车场列表
    TotalSpaces  int              // 总车位
    FreeSpaces   int              // 空闲车位
    TurnoverRate float64          // 周转率
    CreatedAt    time.Time        // 创建时间
    UpdatedAt    time.Time        // 更新时间
}
```

#### Vehicle（车辆）
```go
type Vehicle struct {

```

```go
    OrderID     string    // 订单ID
    CreatedAt   time.Time // 创建时间
    CompletedAt time.Time // 完成时间
}
```

### 6.2 数据关系

```
ParkingPool (1) ─── (N) ParkingLot
ParkingLot (1) ─── (N) ParkingSpace
ParkingSpace (1) ─── (N) ParkingReservation
ParkingReservation (1) ─── (1) ParkingSession
ParkingSession (1) ─── (1) Payment
Vehicle (1) ─── (N) Route
```
