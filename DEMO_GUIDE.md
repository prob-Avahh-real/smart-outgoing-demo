# 智慧交通演示系统 - Demo部署指南

## 系统概述

这是一个完整的智慧交通演示系统，包含以下功能：

### 核心功能
- **智能车辆生成**：一键生成10-30辆互不重叠的智能车辆
- **智慧行车算法**：车辆自动避让、路径优化
- **智能泊车算法**：自动寻位、泊车引导
- **C-V2X模拟**：车路协同通信模拟
- **实时轨迹演示**：车辆沿路径移动，显示轨迹
- **移动端支持**：手机扫码访问，查看高德地图

### 页面功能
- **首页** (`/`) - 主演示页面，车辆轨迹展示
- **Dashboard** (`/dashboard`) - 数据看板，C-V2X统计
- **Parking** (`/parking`) - 停车场管理
- **Payment** (`/payment`) - 支付系统
- **AMap** (`/amap`) - 高德地图集成

## 快速开始

### 1. 环境要求
- Go 1.21+
- 现代浏览器（Chrome、Edge、Safari）
- 网络连接（需要加载高德地图API）

### 2. 启动服务

```bash
# 进入项目目录
cd smart-outgoing-demo

# 启动服务（默认端口8080）
go run cmd/server/main.go

# 或使用编译后的二进制文件
go build -o smart-traffic cmd/server/main.go
./smart-traffic
```

### 3. 访问系统

打开浏览器访问：`http://localhost:8080`

## 演示流程

### 步骤1：启动服务
```bash
go run cmd/server/main.go
```

### 步骤2：打开首页
访问 `http://localhost:8080`，等待地图加载完成

### 步骤3：生成智能车辆
点击 **"创建示例车（10～30）"** 按钮
- 系统会自动检测网络连接
- 生成10-30辆互不重叠的智能车辆
- 车辆分为"智慧行车"（🚗）和"智能泊车"（🅿️）两种类型

### 步骤4：开始演示
点击 **"开始"** 按钮
- 车辆开始沿路径移动
- 智慧行车车辆会自动避让其他车辆
- 智能泊车车辆接近终点时减速
- 不同类型车辆显示不同颜色的轨迹

### 步骤5：移动端体验
点击 **"📱 二维码"** 按钮
- 扫描二维码在手机上打开
- 查看真实地址对应的高德地图

### 步骤6：查看数据看板
点击 **"📊 看板"** 按钮或访问 `/dashboard`
- 查看C-V2X统计数据
- 查看RSU路侧单元信息
- 查看V2X消息记录

## 功能说明

### 智慧行车算法
- **车辆避让**：实时检测前方车辆距离，自动减速避让
- **路径优化**：根据车辆类型调整速度
- **视觉反馈**：蓝色轨迹，🚗图标

### 智能泊车算法
- **自动寻位**：车辆自动驶向指定终点
- **泊车引导**：接近终点时减速（50%进度减速至70%，70%进度减速至50%）
- **视觉反馈**：绿色轨迹，🅿️图标

### 网络检测
- 所有操作前自动检测网络连接
- 网络断开时阻止操作并提示
- 网络弱时警告但允许继续

### 测试框架
```bash
# 运行所有功能测试
./tests/run_tests.sh
```

## 配置说明

### 环境变量
创建 `.env` 文件：
```bash
ADMIN_TOKEN=demo_admin_token
AMAP_JS_KEY=your_amap_key
AMAP_SECURITY_CODE=your_amap_security_code
```

### 高德地图配置
- 确保高德地图API密钥有效
- 检查网络连接（需要访问 `webapi.amap.com`）
- 如果加载失败，系统会自动重试多个CDN源

## 故障排除

### 地图加载失败
1. 检查网络连接
2. 确认高德地图API密钥有效
3. 尝试刷新页面
4. 检查浏览器控制台错误信息

### 车辆未生成
1. 检查控制台日志
2. 确认网络连接正常
3. 检查服务器是否运行
4. 查看浏览器控制台错误信息

### 二维码无法扫描
1. 确保手机和电脑在同一网络
2. 检查服务器端口是否正确
3. 确认二维码生成成功

## API端点

### 车辆管理
- `GET /api/vehicles` - 获取车辆列表
- `POST /api/vehicles` - 创建车辆
- `DELETE /api/vehicles` - 删除所有车辆

### C-V2X模拟
- `GET /api/cv2x/vehicles` - 获取V2X车辆
- `GET /api/cv2x/rsus` - 获取RSU列表
- `GET /api/cv2x/statistics` - 获取统计数据
- `GET /api/cv2x/messages` - 获取V2X消息

### 模拟控制
- `POST /api/simulation/vehicles/generate` - 生成随机车辆
- `POST /api/simulation/scenario/:type` - 生成交通场景
- `DELETE /api/simulation/vehicles` - 清空所有车辆

### 其他
- `GET /api/config` - 获取配置
- `GET /api/app/url` - 获取应用URL
- `WS /ws` - WebSocket连接
